Estilos globais para redefinição de CSS, tipografia e links

Requer doctype HTML5

Bootstrap faz uso de elementos HTML e propriedades CSS que requerem o uso do doctype HTML5. Certifique-se de incluí-lo no início de cada página Bootstrapped do seu projeto. 

<!DOCTYPE html>
<html lang="pt-br">
    ...
</html>

Tipografia e links

Dentro do arquivo scaffolding.less, definimos estilos básicos de exibição global, tipografia e link. Especificamente, nós:

  • Remover margem no body
  • Definir background-color: white; no body
  • Use os atributos @baseFontFamily, @baseFontSize, e @baseLineHeight como nossa base tipográfica
  • Defina a cor do link global via @linkColor e aplique sublinhados de link apenas em :hover

Redefinir via Normalize

A partir do Bootstrap 2, a redefinição CSS tradicional evoluiu para fazer uso de elementos do Normalize.css , um projeto que também alimenta o HTML5 Boilerplate .

A nova redefinição ainda pode ser encontrada em reset.less, mas com muitos elementos removidos por questão de brevidade e precisão.

Colunas de aninhamento

Com o sistema de régua estático (não fluido) do Bootstrap, o aninhamento é fácil. Para aninhar seu conteúdo, basta adicionar um novo .row e conjunto de colunas .span* dentro de uma coluna .span* existente.

Exemplo

As linhas aninhadas devem incluir um conjunto de colunas que somam o número de colunas da camada mais alta. Por exemplo, duas colunas .span3 aninhadas deverá ser colocado dentro de um .span6.

Porcentagens, sem pixels

O sistema de régua fluida usa porcentagens para larguras de coluna em vez de pixels fixos. Ele também possui as mesmas variações de resposta do nosso sistema de régua fixa, garantindo proporções adequadas para as principais resoluções de tela e dispositivos.

Linhas fluidas

Torne qualquer linha fluida simplesmente alterando .row para .row-fluid. As colunas permanecem exatamente as mesmas, tornando muito fácil alternar entre layouts fixos e fluidos.

Marcação 

<div class="row-fluid">
    <div class="span4">
        ...
    </div>

    <div class="span8">
        ...
    </div>
</div>

Nidificação fluida

O aninhamento com réguas fluidas é um pouco diferente: o número de colunas aninhadas não precisa corresponder a camada mais alta. Em vez disso, suas colunas são redefinidas em cada nível porque cada linha ocupa 100% da coluna da camada mais alta.

@gridColumnWidth 60px Largura de cada coluna
@gridGutterWidth 20px Espaço negativo entre colunas
@siteWidth Soma calculada de todas as colunas e gutters Conta o número de colunas e gutters para definir a largura do mixin .container-fixed()

Variáveis em LESS

Integradas ao Bootstrap estão algumas variáveis para personalizar o sistema de régua padrão de 940px, documentado acima. Todas as variáveis da régua são armazenadas em variables.less.

Como personalizar

Modificar a régua significa mudar as três variáveis @grid* e recompilando o Bootstrap. Altere as variáveis da régua em variables.less e use uma das quatro maneiras documentadas para recompilar. Se você estiver adicionando mais colunas, certifique-se de adicionar o CSS para aquelas em grid.less.

Permanecendo responsivo

A personalização da régua funciona apenas no nível padrão, a régua de 940px. Para manter os aspectos responsivos do Bootstrap, você também terá que personalizar as réguas em responsive.less.

Layouts Templates básicos para criar páginas da web

Layout fixo

O layout centralizado padrão e simples de 940 pixels de largura para praticamente qualquer site ou página fornecido por um único <div class="container">. 

<body>
    <div class="container">
        ...
    </div>
</body>

Layout fluido

<div class="container-fluid"> oferece estrutura de página flexível, larguras mínima e máxima e uma barra lateral esquerda. É ótimo para aplicativos e documentos. 

<div class="container-fluid">
    <div class="row-fluid">
        <div class="span2">
            <!--
              - Conteúdo da sidebar.
             -->
        </div>

        <div class="span10">
            <!--
              - Conteúdo do body.
             -->
        </div>
    </div>
</div>

Design responsivo Consultas de mídia para vários dispositivos e resoluções

O que eles fazem

As consultas de mídia permitem CSS personalizado com base em uma série de condições - proporções, larguras, tipo de exibição, etc. - mas geralmente se concentram em min-width e max-width.

  • Modifique a largura da coluna em nossa régua
  • Empilhe elementos em vez de flutuar sempre que necessário
  • Redimensione os títulos e o texto para serem mais adequados aos dispositivos

Use consultas de mídia com responsabilidade e apenas como ponto de partida para seu público móvel. Para projetos maiores, considere bases de código dedicadas e não camadas de consultas de mídia.

Dispositivos suportados

Bootstrap oferece suporte a várias consultas de mídia em um único arquivo para ajudar a tornar seus projetos mais apropriados em diferentes dispositivos e resoluções de tela. Aqui está o que está incluído:

Label Largura do layout Largura do column Largura do gutter
Smartphones 480px e abaixo Colunas fluidas, sem larguras fixas
Smartphones para tablets 767px e abaixo Colunas fluidas, sem larguras fixas
Tablets em retrato 768px e acima 42px 20px
Padrão 980px para cima 60px 20px
Display grande 1200px para cima 70px 30px

Requer metatag

Para garantir que os dispositivos exibam páginas responsivas corretamente, inclua a meta tag da janela de visualização. 

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Usando as media queries

O Bootstrap não inclui automaticamente essas consultas de mídia, mas entendê-las e adicioná-las é muito fácil e requer configuração mínima. Você tem algumas opções para incluir os recursos responsivos do Bootstrap:

  1. Use a versão responsiva compilada, bootstrap-responsive.css
  2. Adicionar @import "responsive.less" e recompilar o Bootstrap
  3. Modifique e recompile responsive.less como um arquivo separado

Por que não apenas incluí-lo ? Verdade seja dita, nem tudo precisa ser responsivo. Em vez de encorajar os desenvolvedores a remover esse recurso, achamos melhor ativá-lo. 

//
// Telefones em paisagem e para baixo.
//
@media (max-width: 480px)
{
    ...
}

//
// Telefone paisagem para tablet retrato.
//
@media (max-width: 768px)
{
    ...
}

//
// Retrato do tablet para paisagem e desktop.
//
@media (min-width: 768px) and (max-width: 980px)
{
    ...
}

//
// Desktop grande.
//
@media (min-width: 1200px)
{
    ...
}

Classes utilitárias responsivas

O que eles são

Para um desenvolvimento mais rápido e compatível com dispositivos móveis, use essas classes utilitárias básicas para mostrar e ocultar conteúdo por dispositivo.

Quando usar

Use de forma limitada e evite criar versões totalmente diferentes do mesmo site. Em vez disso, use-os para complementar a apresentação de cada dispositivo.

Por exemplo, você pode mostrar um elemento <select> para navegação em layouts móveis, mas não em tablets ou desktops.
