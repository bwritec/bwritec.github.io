Grupos de botões Junte-se aos botões para obter mais funcionalidades semelhantes às da barra de ferramentas

Grupos de botões

Use grupos de botões para unir vários botões como um componente composto. Construa-os com uma série de elementos <a> ou <button>.

Melhores Práticas

Recomendamos as seguintes diretrizes para usar grupos de botões e toolbars:

  • Sempre use o mesmo elemento em um único grupo de botões, <a> ou <button>.
  • Não misture botões de cores diferentes no mesmo grupo de botões.
  • Use ícones além ou em vez de texto, mas certifique-se de incluir texto alternativo e de título quando apropriado.

Relacionado Grupos de botões com menus suspensos (veja abaixo) devem ser chamados separadamente e sempre incluir um cursor suspenso para indicar o comportamento pretendido.

Exemplo padrão

Veja como o HTML procura um grupo de botões padrão criado com botões de tag âncora: 

<div class="btn-group">
    <button class="btn">
        1
    </button>

    <button class="btn">
        2
    </button>

    <button class="btn">
        3
    </button>
</div>

Exemplo de toolbar

Combine conjuntos de <div class="btn-group"> dentro de <div class="btn-toolbar"> para componentes mais complexos. 

<div class="btn-toolbar">
    <div class="btn-group">
        ...
    </div>
</div>

Padrões leves Mesma marcação, classes diferentes

Classe base

Todos os componentes de navegação aqui - tabs, pills e listas - compartilham a mesma marcação e estilos básicos através da classe .nav.

Quando usar

Alterar entre eles é fácil e requer apenas alterar muito pouca marcação.

            <p>
                Estou na Seção 1.
            </p>
        </div>

        <div class="tab-pane" id="2">
            <p>
                Olá, estou na Seção 2.
            </p>
        </div>
    </div>
</div>

A navbar requer apenas alguns divs para estruturá-la bem para exibição estática ou fixa. 

<div class="navbar">
    <div class="navbar-inner">
        <div class="container">
            ...
        </div>

Navbar fixa

Fixe a navbar na parte mais alta ou baixa da janela de visualização com uma classe adicional na div mais externa, .navbar. 

<div class="navbar navbar-fixed-top">
    ...
</div>
<div class="navbar navbar-fixed-bottom">
    ...
</div>

Ao conectar a navbar, lembre-se de levar em consideração o espaço oculto abaixo. Adicione 40px ou mais de adição ao <body>. Certifique-se de adicionar isso após o CSS principal do Bootstrap e antes do CSS responsivo opcional.

Nome da brand

Um link simples para mostrar sua marca ou nome de projeto requer apenas uma tag âncora. 

<a class="brand" href="#">
    Nome do projeto
</a>

Formulários na navbar

Para estilizar e posicionar corretamente um formulário na navbar, adicione as classes apropriadas conforme mostrado abaixo. Para um formulário padrão, inclua .navbar-form e também .pull-left ou .pull-right para alinhá-lo adequadamente. 

<form class="navbar-form pull-left">
    <input type="text" class="span2">
</form>

Para um formulário de pesquisa mais personalizado, adicione a classe .navbar-search para receber estilos especializados na navbar. 

<form class="navbar-search pull-left">
    <input type="text" class="search-query" placeholder="Buscar">
</form>

Variação responsiva opcional

Dependendo da quantidade de conteúdo na sua topbar, você pode querer implementar as opções responsivas. Para fazer isso, envolva seu conteúdo de navegação em um div contendo, .nav-collapse.collapse, e adicione o botão de alternância da navbar, .btn-navbar. 

<div class="navbar">
    <div class="navbar-inner">
        <div class="container">
            <!--
              - .btn-navbar é usado como alternância para o conteúdo
              - recolhido da navbar.
             -->
            <a class="btn btn-navbar" data-toggle="collapse" data-target=".nav-collapse">
                <span class="icon-bar"></span>
                <span class="icon-bar"></span>
                <span class="icon-bar"></span>
            </a>

            <!--
              - Certifique-se de deixar a marca lá fora se quiser
              - que ela seja mostrada.
             -->
            <a class="brand" href="#">
                Nome do projeto
            </a>

            <!--
              - Tudo o que você deseja ocultar em 940px ou menos,
              - coloque aqui.
             -->
            <div class="nav-collapse">
                <!--
                  - .nav, .navbar-search, .navbar-form, etc.
                 -->
            </div>
        </div>
    </div>
</div>
Atenção ! A navbar responsiva requer o plugin collapse.

Pager Para links rápidos anteriores e seguintes

Sobre o pager

O componente pager é um conjunto de links para implementações de paginação simples com marcação leve e estilos ainda mais leves. É ótimo para sites simples, como blogs ou revistas.

Estado desativado opcional

Links de pager também usam a classe .disabled geral da paginação.

Exemplo padrão

Por padrão, o pager centraliza os links.

  • Anterior
  • Próximo 
<ul class="pager">
    <li>
        <a href="#">
            Anterior
        </a>
    </li>

    <li>
        <a href="#">
            Próximo
        </a>
    </li>
</ul>

Links alinhados

Como alternativa, você pode alinhar cada link nas laterais:

  • ← Primeiro
  • Último → 
<ul class="pager">
    <li class="previous">
        <a href="#">
            &larr; Primeiro
        </a>
    </li>

    <li class="next">
        <a href="#">
            Último &rarr;
        </a>
    </li>
</ul>

Componentes tipográficos. Título da página e unidade principal para segmentação de conteúdo

Unidade hero

Bootstrap fornece um componente leve e flexível chamado unidade hero para exibir o conteúdo do seu site. Funciona bem em sites de marketing e com muito conteúdo.

Marcação

Envolva seu conteúdo em um div igual a: 

<div class="hero-unit">
    <h1>
        Título
    </h1>

    <p>
        Tagline
    </p>

    <p>
        <a class="btn btn-primary btn-large">
            Saber mais
        </a>
    </p>
</div>

Olá, mundo !

Esta é uma unidade de hero simples, um componente simples no estilo jumbotron para chamar atenção extra para o conteúdo ou informação em destaque.

Título da página

Um shell simples para um h1 para espaçar e segmentar adequadamente seções de conteúdo em uma página. Ele pode utilizar os elementos h1 padrão small, bem como a maioria dos outros componentes (com estilos adicionais).

Exemplo de título da página Subtexto para título 

<div class="page-header">
    <h1>
        Exemplo de título da página
    </h1>
</div>

Thumbnails Réguas de imagens, vídeos, texto e muito mais

Thumbnails padrão

Por padrão, as miniaturas do Bootstrap são projetadas para mostrar imagens vinculadas com o mínimo de marcação necessária.

Altamente personalizável

Com um pouco de marcação extra, é possível adicionar qualquer tipo de conteúdo HTML, como títulos, parágrafos ou botões, às thumbnails.

Por que usar thumbnails

Thumbnails (anteriormente .media-grid até a versão 1.4) são ótimos para réguas de fotos ou vídeos, resultados de pesquisa de imagens, produtos de varejo, portfólios e muito mais. Podem ser links ou conteúdo estático.

Marcação simples e flexível

Marcação de thumbnail é simples - um ul com qualquer número de elementos li é tudo o que é necessário. Também é super flexível, permitindo qualquer tipo de conteúdo com um pouco mais de marcação para agrupar seu conteúdo.

Usa tamanhos de coluna da régua

Por último, o componente de miniaturas usa classes de sistema de régua existentes - como .span2 ou .span3 - para controle das dimensões das thumbnails.

A marcação

Conforme mencionado anteriormente, a marcação necessária para miniaturas é leve e direta. Aqui está uma olhada na configuração padrão para imagens vinculadas : 

<ul class="thumbnails">
    <li class="span3">
        <a href="#" class="thumbnail">
            <img src="assets/img/260x180.png" alt="">
        </a>
    </li>

    ...
</ul>

Para conteúdo HTML personalizado em miniaturas, a marcação muda ligeiramente. Para permitir conteúdo em nível de bloco em qualquer lugar, trocamos o <a> para um <div> igual a: 

<ul class="thumbnails">
    <li class="span3">
        <div class="thumbnail">
            <img src="assets/img/260x180.png" alt="">

            <h5>
                Label da thumbnail
            </h5>

            <p>
                Legenda da thumbnail aqui...
            </p>
        </div>
    </li>

    ...
</ul>

Alertas Estilos para mensagens de sucesso, aviso e erro

Padrões leves

Classe base reescrita

Com o Bootstrap 2, simplificamos a classe base: .alert em vez de .alert-message. Também reduzimos a marcação mínima exigida - <p> não é obrigatório por padrão, apenas o exterior <div>.

Mensagem de alerta única

Para um componente mais durável e com menos código, removemos a aparência diferenciadora para alertas de bloco, mensagens que vêm com mais preenchimento e normalmente mais texto. A classe também mudou para .alert-block.

Funciona muito bem com javascript

Bootstrap vem com um ótimo plugin jQuery que suporta mensagens de alerta, tornando rápido e fácil descartá-las.

Obter o plugin »

Progress bars Para carregamento, redirecionamento ou status de ação

Exemplos e marcação

Básico

Barra de progresso padrão com gradiente vertical. 

<div class="progress">
    <div class="bar" style="width: 60%;"></div>
</div>

Striped

Usa um gradiente para criar um efeito listrado (sem IE). 

<div class="progress progress-striped">
    <div class="bar" style="width: 20%;"></div>
</div>

Animado

Pega o exemplo listrado e o anima (sem IE). 

<div class="progress progress-striped active">
    <div class="bar" style="width: 40%;"></div>
</div>

Opções e suporte ao navegador

Cores adicionais

As barras de progresso usam alguns dos mesmos botões e classes de alerta para estilos consistentes.

Barras listradas

Semelhante às cores sólidas, temos barras de progresso listradas variadas.

Comportamento

As barras de progresso usam transições CSS3, portanto, se você ajustar dinamicamente a largura via javascript, ela será redimensionada suavemente.

Se você usar a classe .active, seu .progress-striped barra de progresso animarão as listras da esquerda para a direita.

Suporte ao navegador

As barras de progresso usam gradientes, transições e animações CSS3 para obter todos os seus efeitos. Esses recursos não são suportados no IE7-9 ou em versões anteriores do Firefox.

Opera e IE não suportam animações no momento.
