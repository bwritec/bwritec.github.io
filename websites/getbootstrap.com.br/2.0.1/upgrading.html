Atualizando para Bootstrap 2

Aprenda sobre mudanças e adições significativas desde a versão 1.4 com este guia prático.

Mudanças no projeto

  • Documentos: atualizações importantes em toda a estrutura geral, exemplos e trechos de código. Também responsivo com novas consultas de mídia.
  • Documentos: todas as páginas de documentos agora são alimentadas por modelos Moustache e as strings são agrupadas em tags i18n para tradução pelo Centro de Tradução. Todas as alterações na documentação devem ser feitas aqui e depois compiladas (semelhante ao nosso CSS e LESS).
  • Estrutura de diretório do repositório: removeu o CSS compilado da raiz em favor de um grande link de download direto na página inicial dos documentos. CSS compilado está em /docs/assets/css/.
  • Documentos e repositório: um makefile, basta digitar make no Terminal e obtenha documentos e CSS atualizados.

Scaffolding

Sistema de régua

  • Sistema de régua atualizado, agora com apenas 12 colunas em vez de 16.
  • A abordagem responsiva significa que seus projetos funcionam praticamente imediatamente em smartphones, tablets e muito mais.
  • Removido o suporte de colunas de régua não utilizadas (por padrão) para 17 a 24 colunas.

Responsivo (media queries)

  • Consultas de mídia adicionadas para suporte básico em dispositivos móveis e tablets.
  • CSS responsivo é compilado separadamente, como bootstrap-responsive.css.

CSS básico

Tipografia

  • Elementos h4 foram reduzidos de 16px para 14px com um padrão de 18px da line-height.
  • Elementos h5 foram reduzidos de 14px para 12px.
  • Elementos h6 foram reduzidos de 13px para 11px.
  • Opção alinhada à direita para aspas se float: right;.

Código

  • Novo estilo gráfico para <code>
  • Estilos Google Code Prettify atualizado (com base nas essências do Git).

Tabelas

  • Suporte aprimorado para colspan e rowspan
  • Estilos agora restritos à nova classe base, .table
  • Classes de tabela padronizadas com .table- obrigatório como prefixo
  • Removidas opções de cores de tabela não utilizadas (muito código para tão pouco impacto).
  • Suporte removido para TableSorter.

Botões

  • Novas classes de cores e tamanhos, todas prefixadas com .btn-.
  • IE9: removeu gradientes e adicionou cantos arredondados.
  • Estado ativo atualizado para tornar o estilo mais claro em grupos de botões (novo) e ficar melhor com a transição personalizada.
  • Novo mixin, .buttonBackground, para definir gradientes de botão.
  • A classe .secondary foi removido dos exemplos modais em nossos documentos porque nunca teve estilos associados.

Formulários

  • O estilo de formulário padrão agora é vertical (empilhado) para usar less CSS e adicionar maior flexibilidade.
  • Classe de formulário padronizadas com .form- obrigatório como prefixo.
  • Novos padrões de formulário integrados para formulários de pesquisa, inline e horizontais.
  • Para formulários horizontais, classes anteriores .clearfix e .input são equivalentes ao novo .control-group e .controls.
  • Marcação de formulário horizontal mais flexível com classes para todos os estilos, incluindo nova classe opcional para o label.
  • Estados do formulário: cores atualizadas e personalizáveis através de novas variáveis LESS.

Ícones, por Glyphicons

  • Novo conjunto de ícones Glyphicons Halflings adicionado em forma de sprite, em preto e branco.
  • Marcação simples necessária para um ícone em vários contextos: <i class="icon-cog"></>
  • Adicionar outra classe, .icon-white, para variação branca do mesmo ícone.

Componentes

Grupo de botões e dropdowns

  • Dois novos componentes no 2.0: grupos de botões e menus dropdowns de botões.
  • Dependência: os menus dropdown de botões são criados em grupos de botões e, portanto, exigem todos os seus estilos.
  • Grupo de botões, .btn-group, pode ser agrupado um nível acima com uma toolbar de botões, .btn-toolbar.

Navegação

  • Tabs e pills agora exigem o uso de uma nova classe base, .nav, em seus <ul> e os nomes das classes agora são .nav-pills e .nav-tabs.
  • Adicionada nova variação da lista de navegação que usa a mesma classe base, .nav.
  • Foram adicionadas tabs e pills verticais - basta adicionar .nav-stacked para o <ul>
  • Os pills foram reestilizados para serem menos arredondados por padrão.
  • Os pills agora têm suporte para menu dropdown (eles compartilham a mesma marcação e estilos das tabs).

Navbar (antigamente topbar)

  • Classe base alterada de .topbar para .navbar.
  • Agora suporta posição estática (comportamento padrão, não fixo) e fixada no topo da janela de visualização via .navbar-fixed-top (anteriormente suportado apenas fixo).
  • Adicionados divisores verticais à navegação de nível mais alto.
  • Suporte aprimorado para formulários inline na navbar, que agora exigem .navbar-form para definir o escopo adequado dos estilos apenas para os formulários pretendidos.
  • O formulário de busca da navbar agora requer o uso da classe .navbar-search e sua entrada o uso de .search-query. Para posicionar o formulário de pesquisa, você deve usar .pull-left ou .pull-right.
  • Adicionada marcação responsiva opcional para recolher o conteúdo da navbar para resoluções e dispositivos menores. Consulte o manual da navbar para saber como utilizar.

Menus Dropdown

  • Atualizado o .dropdown-menu para aumentar o espaçamento.
  • Agora requer que você adicione um <span class="caret"></span> para mostrar a seta no dropdown.
  • Agora requer que você adicione um atributo data-toggle="dropdown" para obter comportamento de alternância.
  • O navbar (topbar fixa) possui novos menus dropdowns. As versões escuras se foram e em seu lugar estão as brancas padrão com um sinal de intercalação adicional na parte alta para maior clareza de posição.

Labels

  • Cores dos rótulos atualizadas para corresponder às cores do estado do formulário.
  • Eles não apenas correspondem graficamente, mas também são alimentados pelas mesmas novas variáveis.

Thumbnails

  • Anteriormente .media-grid, agora apenas .thumbnails, estendemos completamente esse componente para mais usos, mantendo a simplicidade geral pronta para uso.
  • As thumbnails individuais agora exigem a classe .thumbnail.

Alertas

  • Nova classe base: .alert em vez de .alert-message
  • Nomes de classes padronizados para outras opções, agora todos começando com .alert-
  • Estilos de alerta base reprojetados para combinar os alertas padrão e os alertas em nível de bloco em um só.
  • Classe de alerta em nível de bloco alterada: .alert-block em vez de .block-message

Progress bars

  • Novo em 2.0.
  • Apresenta vários estilos por meio de classes, incluindo variações listradas e animadas via CSS3.

Componentes diversos

  • Adicionada documentação para o componente de well e o ícone de fechamento (usado em modais e alertas).

Plugins Javascript

Atenção ! Reescrevemos quase tudo para nossos plugins, então vá para a página Javascript para ler mais.

Tooltips

  • O método do plugin foi renomeado de twipsy() para tooltip(), e o nome da classe mudou de twipsy para tooltip.
  • O valor da opção placement, aquilo que foi below agora é bottom, e above agora é top.
  • A opção animate foi renomeado para animation.
  • A opção html foi removido, já que o padrão das tooltips agora é permitir HTML.

Popovers

  • Elementos da sub camada agora com namespace adequado: .title para .popover-title, .inner para .popover-inner, e .content para .popover-content.
  • O valor da opção placement, aquilo que foi below agora é bottom, e above agora é top.

Novos plugins

  • Collapse
  • Carousel
  • Typeahead
